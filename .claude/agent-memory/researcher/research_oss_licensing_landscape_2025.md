# OSS Licensing Landscape for Infrastructure/Developer Tools (2025-2026)

**Why:** Informing licensing decisions for vexa-agentic-runtime. The infra/devtools space has undergone massive licensing upheaval 2018-2025 and lessons are now clear.

**How to apply:** Use this as the decision framework when choosing a license. The key lesson from 2018-2025: companies that started permissive and switched restrictive lost more than they gained. Choose deliberately from day one.

## Full analysis delivered in conversation on 2026-03-26.

Key conclusions:
1. AGPL + dual-license is the emerging winner for infra projects that need cloud protection
2. FSL is the cleanest "source available" option if AGPL is too aggressive
3. MIT/Apache-2.0 remain best for maximum adoption but zero cloud protection
4. BSL/SSPL are losing favor — both Redis and Elastic reversed course to AGPL
5. "Fair Source" is gaining legitimacy as a formal third category
