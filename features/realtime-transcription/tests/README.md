# Realtime Transcription Tests

## What's tested

The core transcription pipeline: `SpeakerStreamManager` ‚Üí `TranscriptionClient` ‚Üí Whisper ‚Üí confirmed segments.

No meetings, no bots, no browsers. Just audio in, segments out.

## Tests

### Unit tests (`speaker-streams.test.ts`)

Mocked Whisper responses. Tests the SpeakerStreamManager algorithm:
1. **Offset advancement** ‚Äî after confirmation, Whisper receives only unconfirmed audio
2. **Buffer continuity** ‚Äî no full reset on confirmation, submissions stay small
3. **Speaker change flush** ‚Äî emits on `flushSpeaker()`
4. **Short segment skip** ‚Äî <2s audio kept for next turn on flush
5. **Buffer trim** ‚Äî confirmed audio trimmed at max size

### Pipeline tests (`speaker-streams.wav-test.ts`)

Real audio through real Whisper on GPU. Feeds a WAV file chunk-by-chunk into the pipeline, logs every event:
- `SUBMIT` ‚Äî audio sent to Whisper (shows duration)
- `RESULT` ‚Äî Whisper response (shows latency, language, text)
- `CONFIRMED` ‚Äî segment confirmed and emitted (shows time range, text)
- `FLUSH` ‚Äî speaker change or end-of-audio flush

### Test audio files (`audio/`)

Generated by Piper TTS:
- `short-sentence.wav` ‚Äî ~3s, single sentence
- `medium-paragraph.wav` ‚Äî ~14s, three sentences
- `long-monologue.wav` ‚Äî ~40s, extended speech
- `two-speakers.wav` ‚Äî ~15s, simulated dialogue (single voice, tests buffer behavior)

## How to run

```bash
# From repo root
cd features/realtime-transcription

# Generate test audio (needs tts-service container)
bash tests/generate-test-audio.sh

# Run all tests (unit + pipeline)
bash tests/run-pipeline-test.sh

# Run one test
bash tests/run-pipeline-test.sh short-sentence
bash tests/run-pipeline-test.sh long-monologue

# Run against custom WAV
bash tests/run-pipeline-test.sh /path/to/your/file.wav
```

## Prerequisites

- `transcription-service` running on port 8085 (`make all` from `deploy/compose/`)
- `vexa-restore-tts-service-1` container for generating test audio
- Node.js with `ts-node` available

## What to look for in the logs

**Healthy pipeline:**
```
[0.0s] [SUBMIT] 2.0s audio ‚Üí Whisper (call #1)
[0.2s] [RESULT] 189ms | lang=en | "Hello everyone, I want to start by"
[0.3s] [SUBMIT] 4.1s audio ‚Üí Whisper (call #2)
[0.5s] [RESULT] 175ms | lang=en | "Hello everyone, I want to start by reviewing our product metrics from last"
[0.5s] [CONFIRMED] 0.0s-0.5s | "Hello everyone, I want to start by reviewing our product metrics from last"
```

- Submission sizes should alternate: grow 2s‚Üí4s, confirm, reset to 2s‚Üí4s, confirm
- After `CONFIRMED`, next `SUBMIT` should be small (2s, not growing from 4s)
- `Avg Whisper time` should be <500ms on GPU

**Broken pipeline (what we fixed):**
- Submissions growing: 2s‚Üí4s‚Üí6s‚Üí8s‚Üí10s‚Üí... (not trimming confirmed audio)
- Same text confirmed repeatedly (idle resubmit loop)
- Ghost segments ("significantly", "Thank you.") ‚Äî hallucinated from silence
