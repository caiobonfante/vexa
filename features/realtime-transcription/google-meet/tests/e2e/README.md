# Google Meet E2E Transcription Tests

End-to-end validation of the full pipeline:
TTS bots speak scripted utterances into a live Google Meet, audio is captured per-speaker, speaker identity locks via voting, Whisper transcribes, segments flow through Redis to the collector, and persist to Postgres. Ground truth is exact because we wrote the script.

## Test Scripts

| Script | Utterances | Coverage |
|--------|-----------|----------|
| `test-e2e.sh` | 9 | Basic — short sentences, 3 speakers, clean turn-taking |
| `test-e2e-stress.sh` | 20 | Stress — long monologues, technical jargon, short interjections |

Both scripts send TTS bots to speak, wait for pipeline completion, then dump output segments.

## Scorer

`score-e2e.py` (in this directory) matches ground-truth utterances to output segments via text similarity (difflib), then computes:

- **Speaker accuracy** — % of matched segments attributed to the correct speaker
- **WER** — word error rate across all matched segments
- **Completeness** — % of GT utterances that matched an output segment

## Pass Criteria

| Metric | Threshold |
|--------|-----------|
| Speaker accuracy | >= 90% |
| Completeness | >= 80% |
| WER | <= 30% |

## Usage

```bash
# Basic test (requires a live Google Meet URL)
./test-e2e.sh --meeting "https://meet.google.com/xxx-yyyy-zzz"

# Stress test
./test-e2e-stress.sh --meeting "https://meet.google.com/xxx-yyyy-zzz"

# Score results
python3 score-e2e.py results/<run-dir>/
```

Ensure the compose stack is running (`make all` from `deploy/compose/`) before starting.

## Development Notes

### Test Mechanics

**Ground truth format:**
```json
[
  {"speaker": "Alice", "text": "Exact words spoken via TTS", "pause": 12},
  ...
]
```
The `pause` field controls seconds to wait after sending the TTS command before the next utterance.

**Session UID extraction** (needed to query Postgres):
1. Parse from recorder bot logs: `grep -oP 'session_uid[=: ]+\K[a-f0-9-]+'`
2. Query `meeting_sessions` table: `SELECT session_uid FROM meeting_sessions WHERE meeting_id=$RECORDER_ID`

**DB query:**
```sql
SELECT segment_id, speaker, text, start_time, end_time, language, created_at
FROM transcriptions
WHERE session_uid = '{session_uid}'
ORDER BY start_time;
```

### Timing

- 20s after bots join: audio capture initialization
- 8-12s between utterances: enough for TTS playback + transcription
- 30s after last utterance: transcription confirmation buffer
- 35s more: Postgres immutability threshold (collector holds 30s before persisting)
- Total runtime: ~4 minutes

### Matching Algorithm

A single GT utterance may appear as:
- 1 segment (exact match)
- 2-3 segments (Whisper split at sentence boundaries)
- Merged with adjacent content (Whisper combined short utterances)

The scorer tries individual segments, consecutive pairs, and triples, picking the highest similarity match. Uses `SequenceMatcher` for text similarity and word-level Levenshtein for WER. Text is normalized: lowercase, strip punctuation, collapse whitespace.

### Results Directory Structure

```
results/e2e-YYYY-MM-DD-HHMMSS/
  meeting-id.txt        # Google Meet code
  session-uid.txt       # Session UUID for DB queries
  ground-truth.json     # TTS script (generated by test)
  recorder-full.log     # Complete recorder bot logs
  confirmed-segments.log # Grep of CONFIRMED lines
  bot-segments.json     # Parsed confirmed segments
  db-segments.csv       # Postgres export
  score.txt             # Human-readable report
  score.json            # Machine-readable results
```
