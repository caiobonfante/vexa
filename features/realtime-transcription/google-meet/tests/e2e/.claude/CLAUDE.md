# E2E Pipeline Test — Agent Instructions

## What this tests

Full pipeline: TTS bot audio → Google Meet → Recorder bot audio capture → Speaker identity → SpeakerStreamManager → Whisper → SegmentPublisher → Redis → Collector → Postgres.

Ground truth is the TTS script — we know exactly what each speaker said.

## How to run

```bash
# Prerequisites: same as speaker-voting tests (see ../speaker-voting/.claude/CLAUDE.md)
# 1. Create meeting + auto-admit
# 2. Clean orphaned meetings
# 3. Run:

bash test-e2e.sh --meeting {MEETING_CODE}
```

The test:
1. Sends Recorder bot (transcribe_enabled: true)
2. Sends Alice, Bob, Charlie bots (transcribe_enabled: false, TTS only)
3. Executes ground truth script: 9 utterances, sequential, 8-12s pause between
4. Waits 30s for transcription confirmation
5. Waits 35s more for Postgres immutability threshold
6. Fetches segments from Postgres via `COPY ... TO STDOUT CSV`
7. Scores at two levels: bot-confirmed segments AND DB-persisted segments

## Scoring (score-e2e.py)

For each ground truth utterance, finds the best matching output segment(s):
- Tries individual segments, consecutive pairs, and triples (handles Whisper splitting)
- Uses `SequenceMatcher` for text similarity and word-level Levenshtein for WER
- Text is normalized: lowercase, strip punctuation, collapse whitespace

**Pass criteria:**
- Speaker accuracy >= 90%
- Completeness >= 80% (found/total)
- WER <= 30%

## Ground truth format

```json
[
  {"speaker": "Alice", "text": "Exact words spoken via TTS", "pause": 12},
  ...
]
```

The `pause` field controls seconds to wait after sending the TTS command before the next utterance.

## Key implementation details

### Session UID

The test needs the `session_uid` to query Postgres. It tries two methods:
1. Parse from recorder bot logs: `grep -oP 'session_uid[=: ]+\K[a-f0-9-]+'`
2. Query `meeting_sessions` table: `SELECT session_uid FROM meeting_sessions WHERE meeting_id=$RECORDER_ID`

### DB query

```sql
SELECT segment_id, speaker, text, start_time, end_time, language, created_at
FROM transcriptions
WHERE session_uid = '{session_uid}'
ORDER BY start_time;
```

### Timing

- 20s after bots join: audio capture initialization
- 8-12s between utterances: enough for TTS playback + transcription
- 30s after last utterance: transcription confirmation buffer
- 35s more: Postgres immutability threshold (collector holds 30s before persisting)
- Total runtime: ~4 minutes

### Matching algorithm

A single GT utterance may appear as:
- 1 segment (exact match)
- 2-3 segments (Whisper split at sentence boundaries)
- Merged with adjacent content (Whisper combined short utterances)

The scorer tries all combinations and picks the highest similarity match.

## Results format

```
results/e2e-YYYY-MM-DD-HHMMSS/
  meeting-id.txt        # Google Meet code
  session-uid.txt       # Session UUID for DB queries
  ground-truth.json     # TTS script (generated by test)
  recorder-full.log     # Complete recorder bot logs
  confirmed-segments.log # Grep of CONFIRMED lines
  bot-segments.json     # Parsed confirmed segments
  db-segments.csv       # Postgres export
  score.txt             # Human-readable report
  score.json            # Machine-readable results
```
